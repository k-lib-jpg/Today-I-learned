#dockerfileとは

カスタマイズされたイメージを準備するための設計書である。コンテナ自体に直接ソフトウェアを入れてもいいが、そうするとコンテナ環境の再現ができないため、イメージのテンプレートが必要。

#dockerfileコマンド

docker image build {ディレクトリパス}
→カレントディレクトリだと.

-tをつけて後ろにつけたい名前を記述すれば名前をつけられる

#dockerのイメージはレイヤー構造になっている

dockerのイメージはレイヤー構造となっており、イメージの変更差分がレイヤーとして積み上げられている構造になっている。そのことで容量の上昇を抑えめにしている。
